# GW9251 ADC 특성평가 자동화

[English](README.en.md)

GW9251(24bit ADC) 특성평가 수작업 흐름(Tera Term으로 DAC 전압 설정 → 레지스터
설정 → 1024샘플 캡처 → 엑셀 리포트에 복사/붙여넣기)을 자동화하는 도구입니다.

- `gw9251_autotest.py` — CLI. 보드 자동 식별, DAC/레지스터 설정, 샘플 캡처,
  CSV 백업 + 엑셀 리포트 기록까지 DUT 1개 사이클을 자동 수행
- `gw9251_gui.pyw` — Tkinter GUI 프런트엔드 (위 모듈을 import해서 사용).
  DAC 코드/샘플 수/DUT 번호/엑셀 파일 선택, 실행 로그, 결과 요약(ENOB·STD·
  정확도), 리포트에 DUT 블록 자동 생성 기능 포함
- `test_gw9251.py` — 하드웨어 없이 도는 회귀 테스트 (`python test_gw9251.py`)

## 하드웨어 구성

- Windows PC ↔ STM32 Nucleo 보드 2개 (ST-Link VCP, VID 0x0483)
- **DAC 보드**: 차동 입력 구동. 부팅 시 칩 선택 메뉴
  (`Usage: gw9121 | gw9241 | gw9251`) → `gw9241` 진입 후 `dac init`,
  `dac set 1 <val>`, `dac set 2 <val>`
- **테스트 보드**: GW9251EVM(DUT) 제어. 부팅하면 바로 `GW9251:` 프롬프트
- DUT 교체 시 보드 전원을 내리므로 매 사이클 USB 재열거 + DAC 재설정 필요
  (스크립트가 자동 처리)

## 설치 / 실행

```
pip install pyserial openpyxl

# CLI: DUT 6번을 test_copy.xlsx에 기록
python gw9251_autotest.py --excel test_copy.xlsx --dut 6

# GUI: 더블클릭 실행 (pythonw), 또는
pythonw gw9251_gui.pyw
```

- 실행 전 **Tera Term 등 시리얼 터미널을 모두 종료**할 것 (COM 포트 독점)
- 엑셀 리포트와 파이썬 파일들은 같은 폴더에 둘 것
- 테스트 중에는 리포트 파일을 열어두지 말 것 (열려 있으면 저장 시 재시도
  다이얼로그가 뜸)

## 동작 요약 (DUT 1개 기준)

1. USB에서 보드 2개 대기 → 펌웨어 프롬프트를 프로빙해서 DAC/테스트 보드 자동
   식별 (설정 파일 불필요)
2. 테스트 보드가 `GW9251:` 프롬프트인지 확인
3. DAC 보드: `exit` → `gw9241` → `dac init` → `dac set 1/2`
4. Internal Short 모드: `wr 0x03 0x02` → `rr`로 검증 → `wr 0x04 0x60` → 검증
   → `rd <n>` 캡처 (기본 1024샘플, 약 11샘플/초 → 약 95초 소요)
5. Channel A 모드: `wr 0x04 0x00`으로 반복
6. CSV 백업 저장 + 엑셀 리포트의 해당 DUT 블록에 기록
7. 전원 OFF → 칩 교체 → 전원 ON → 다음 DUT

## 펌웨어 특이사항 (수정 시 주의)

1. **출력에 NUL 바이트와 ANSI 이스케이프가 섞임** (비표준 `ESC[-1D` 포함).
   수신 데이터는 반드시 `clean()`을 거친 뒤 파싱할 것
2. **줄바꿈은 CR 단독** (`LINE_END = "\r"`). CRLF를 보내면 남은 `\n`이 다음
   명령을 오염시켜 usage만 출력됨
3. **서브메뉴 상태가 시리얼 재접속 후에도 유지됨** (전원 사이클로만 초기화).
   DAC 보드는 `gw9241` 전에 `exit`를 먼저 보냄
4. **칩 선택 확인은 칩 이름으로 하면 안 됨**: 실패 응답(usage)에 모든 칩
   이름이 들어 있고, 성공 응답(서브메뉴)에는 칩 이름이 없음. 서브메뉴 마커
   (`dac` / `9251`)로 확인하고, 응답에 `gw9121`이 있으면 아직 최상위 메뉴
5. **`rd` 스트림은 중간에 수 초 멈출 수 있음**: 캡처 종료 판정은 idle 타임아웃
   (20초 무응답) + 하드 캡(300초). 캡처가 끝나면 종료 사유/경과/최대 공백을
   출력하므로 이상 시 이 수치부터 확인
6. **`rr` 응답 형식**: 에코 후 `RREG:A+D: <주소> <값>` (16진수). 정확히
   파싱함 — `A`/`D` 장식 문자도 유효한 16진수라서 토큰 매칭으로 되돌리면
   오판 위험 있음

## 엑셀 리포트 형식

- 5행: 2열 간격의 병합된 `DUT#n` 헤더
- 10행: 모드 헤더 (`Internal Short (0x04 0x60)` / `Channel A (0x04 0x00)`)
- 11~20행: ENOB / STD / 정확도 수식 (스크립트는 건드리지 않음)
- 22~1045행: 샘플 1024개
- 대상 블록은 5행 헤더를 정규식으로 찾아 동적으로 결정하고, 10행 텍스트를
  교차 확인한 뒤에만 기록. `--dut` 생략 시 첫 번째 빈 블록 자동 선택
- GUI는 요청한 DUT 블록이 없으면 마지막 블록의 서식/수식을 복제해서 새 블록을
  만들어 줌 (기존 값이 있으면 지우기 전에 확인)
